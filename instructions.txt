SECTION 1: BACKEND .ENV CREDENTIALS & CONFIGURATION
================================================================================

Required for: gitvibe-backend/.env file

Please provide or confirm the following:

[ ] GitHub API Token (for GitHub integration)
    - Type: Personal Access Token (PAT) or OAuth token
    - Scope: repo, user (typical for GitHub API access)
    - Format: ghp_* or gho_*
    - Environment variable: GITHUB_TOKEN

[ ] Database Configuration
    - [ ] Database Type: SQLite (local development) or PostgreSQL (production)
    - [ ] ORM Framework: SQLModel/SQLAlchemy
    - [ ] For SQLite: Just a file path (e.g., ./gitvibe.db)
    - [ ] For PostgreSQL:
        - Host/Connection String
        - Port (default: 5432)
        - Username
        - Password
        - Database Name

[ ] Redis Configuration
    - [ ] Redis URL (e.g., redis://localhost:6379/0)
    - [ ] Environment variable: REDIS_URL
    - [ ] Purpose: Caching and session management

[ ] Authentication & Security
    - [ ] JWT Secret Key (for token generation)
    - [ ] Session Secret (for session encryption)
    - [ ] Encryption Key (for sensitive data)
    - [ ] CORS Allowed Origins (list of frontend URLs)

[ ] Google AI Integration
    - [ ] Gemini API Key (from Google AI Studio)
    - [ ] Get it at: https://makersuite.google.com/app/apikey
    - [ ] Environment variable: GEMINI_API_KEY
    - [ ] Purpose: AI-powered resume parsing and analysis

[ ] Server Configuration
    - [ ] Backend Port (default: 8000 for FastAPI)
    - [ ] Python Environment (development / staging / production)
    - [ ] Log Level (debug / info / warn / error)

SECTION 2: LOCAL SYSTEM RUNTIMES REQUIRED (Windows Machine)
================================================================================

Required for: Development environment setup

Please confirm these are installed on your Windows machine:

[ ] Python 3.10 or Higher
    - Installation: https://www.python.org/downloads/
    - Verify: Open PowerShell and run: python --version
    - Note: Set up a virtual environment (.venv) in the gitvibe-backend directory
    - Command: python -m venv .venv
    - Activate: .venv\Scripts\Activate.ps1 (PowerShell) or .venv\Scripts\activate.bat (Command Prompt)

[ ] Node.js & npm
    - Minimum version: 16.x or 18.x LTS (recommend 18.x+)
    - Installation: https://nodejs.org/
    - Verify: Open PowerShell and run: node --version && npm --version
    - Purpose: EXCLUSIVELY for building the React/Vite frontend (gitvibe-frontend)

[ ] Git
    - Installation: https://git-scm.com/download/win
    - Verify: Open PowerShell and run: git --version

[ ] Docker Desktop
    - Installation: https://www.docker.com/products/docker-desktop
    - Verify: Open PowerShell and run: docker --version
    - Purpose: Running Redis container for caching and session management
    - Command to run Redis: docker run --name gitvibe-redis -d -p 6379:6379 redis:latest

[ ] Database (Choose One)
    - [ ] SQLite (for local development - no separate installation needed)
        - File-based database stored in gitvibe.db
    - [ ] PostgreSQL (for production or advanced development)
        - Installation: https://www.postgresql.org/download/windows/
        - Optional: pgAdmin for GUI management
        - Verify: Open PowerShell and run: psql --version

[ ] Optional Development Tools
    - [ ] Visual Studio Code (for editing)
    - [ ] Postman or Insomnia (for API testing)
    - [ ] DBeaver or pgAdmin (for database management)
